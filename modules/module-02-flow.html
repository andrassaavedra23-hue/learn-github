El Flujo de Git
Working → Staging → Repository

Todo en Git pasa por 3 áreas. Cada archivo que modificas viaja por estas zonas antes de quedar guardado permanentemente. Haz clic en cada área para entender qué hace.

🎯 Analogía del envío de paquetes:
Working = Tu cuarto desordenado (archivos sueltos)
Staging = La caja donde empacas lo que quieres enviar
Repository = El paquete cerrado y etiquetado, listo para enviar
